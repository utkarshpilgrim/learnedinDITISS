# Conventions for `variables`

There are few conventions that needs to be followed while writing the code in bash or while shell scripting. This is crucial as it make our code more readable and understandable, writing code and having a paradigm of style is important as it make the developer understand the style and implement.

One such convention is using lowercase letters while declaring the variables and start along with underscore. 

In Bash scripting, the readonly built-in command is used to declare variables as read-only, effectively preventing any further modifications to their values after they are initially assigned. This practice is a crucial aspect of writing robust and predictable scripts, especially when dealing with sensitive information or variables that should not be altered accidentally during script execution.
